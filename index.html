Tutorial

Introduction

Our tool is designed to help screen reader users and others who use keyboard navigation to access the internet conduct accessibility audits of websites. Many features of websites are inaccessible to blind and low-vision users, and our auditing tool is designed to make the process of reporting these problems as convenient and accessible as possible.

How to Report a Usability or Accessibility Problem

When browsing a website using a keyboard or screen reader, individual page elements are targeted with focus. When you are targeting an inaccessible feature of the page with your focus, press [f13] to add an annotation about that element to your report. You can use voice or keyboard input to create this annotation, which can be written in plain language or in technical terms such as a specific WCAG violation. It’s your choice!

Reporting Practice: An Inaccessible Image

Let’s get some practice reporting an inaccessible element. The image below this paragraph is inaccessible because of a lack of descriptive text. Use your keyboard to navigate the focus to the image, then report it by pressing [f13].

Exporting Your Report

When you are ready to export a summary of the problems you found, press [F14] to begin exporting a report. The program automatically annotates the website code with the issues you have uncovered and produces a list of accessibility problems that can be used to improve the website.

We have discovered one accessibility error on this page already: the unlabeled image. Press [F14] to produce a report.

Learning More About Accessibility Standards

While your experiences browsing a site are valuable and important to report, there are also formalized standards such as the Web Content Accessibility Guidelines (WCAG) that codify expectations about website features. To learn more about these guidelines, click here to review the official WCAG documentation.
